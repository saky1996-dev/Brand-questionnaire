    <!-- Application Structure Plan: The application transforms the linear markdown brief into a professional, single-page interactive form. A sticky side-navigation allows users to easily jump between sections, improving on the simple scroll of a document. The structure uses distinct, card-like sections for each part of the brief, making the information architecture clear and digestible. This design was chosen to make the daunting task of filling out a brief feel more organized, engaging, and less overwhelming for the client, guiding them through the process step-by-step in a visually pleasing interface. -->

    <!-- Visualization & Content Choices: Since the source is a template, not a data report, traditional charts are not applicable. Instead, 'visualization' is achieved through UI/UX design. Report Info (e.g., 'Keywords/Visual Cues') -> Goal (Gather aesthetic preferences) -> Presentation Method (Interactive clickable tags) -> Interaction (User clicks to select/deselect keywords) -> Justification (More intuitive and less open-ended than a blank text field, guiding the user towards common concepts) -> Library/Method (Vanilla JS and Tailwind CSS). Checkboxes are used for 'Deliverables' to provide a clear, tappable list of options. The entire application serves as an interactive infographic for the briefing process. -->

                            selectedTags = selectedTags.filter(t => t !== tagValue);
